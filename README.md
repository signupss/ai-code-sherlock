## вңЁ What is AI Code Sherlock?

AI Code Sherlock is a desktop IDE assistant and **visual AI Workflow Constructor** that works like a detective for your codebase. It uses local (Ollama) or cloud AI models to analyze errors, generate surgical patches, and run autonomous improvement pipelines вҖ” without ever rewriting your whole file.

**Key idea:** instead of regenerating entire files, the AI outputs only `[SEARCH_BLOCK] вҶ’ [REPLACE_BLOCK]` patches вҖ” precise, reviewable, and safe.

**v3.0 adds:** a fully visual drag-and-drop **Workflow Constructor** вҖ” build infinite-complexity AI agent pipelines with 50+ node types, browser/desktop automation, conditional logic, loops, and variables. No code required.

## в–¶пёҸ Live Script Execution & Interactive Stdin

Click **в–¶ Run Script** in the toolbar to run the active file (or pick any script). Output streams live into the **рҹ“Ӣ Logs** tab вҖ” color-coded by stream:

| Color | Meaning |
|-------|---------|
| White | stdout |
| Red | stderr |
| Purple | System events (start / finish) |
| Yellow | User input echo (`>> text`) |

### Interactive stdin

While the script is running, an **input field** activates at the bottom of the Logs tab. Type anything and press `Enter` or click `вҸҺ` to send it to the running process вҖ” exactly as if you typed it in a terminal.

Works natively with all language input mechanisms:
- Python `input()`, `sys.stdin.read()`
- Bash `read VAR`
- Node.js `readline` / `process.stdin`
- Ruby `gets`, `$stdin.gets`
- PHP `fgets(STDIN)`
- Go `fmt.Scan`, `bufio.NewReader(os.Stdin)`
- And any other language that reads from stdin

The stdin pipe stays **open for the full duration** of the script. The input field is automatically disabled once the script exits.

### How it works internally

```
Qt main thread          Worker event loop (asyncio)
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ       в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
send_stdin("show\n") вҶ’ asyncio.run_coroutine_threadsafe
                      вҶ’ queue.put("show\n")
                      вҶ’ _stdin_pump() drains queue
                      вҶ’ proc.stdin.write(...)
                      вҶ’ proc.stdin.drain()
```

The queue-based pump runs as a third coroutine alongside the stdout/stderr readers вҖ” **no race conditions, no closed-pipe errors**.

## вңӮпёҸ Patch Format

The AI generates structured SEARCH/REPLACE blocks:

```
[SEARCH_BLOCK]
    result = data[0]["value"]
[REPLACE_BLOCK]
    if not data:
        return None
    result = data[0]["value"]
[END_PATCH]
```

**New file creation** вҖ” empty SEARCH_BLOCK:

```
РЎРҫР·РҙР°СҺ `main.py`:
[SEARCH_BLOCK]
[REPLACE_BLOCK]
import pandas as pd

def main():
    df = pd.read_csv("data.csv")
    print(df.describe())

if __name__ == "__main__":
    main()
[END_PATCH]
```

The engine finds the exact match (or normalized whitespace fallback), validates it's unambiguous, creates a backup, applies the patch, runs syntax check вҖ” all before saving.
