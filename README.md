## ✨ What is AI Code Sherlock?

AI Code Sherlock is a desktop IDE assistant that works like a detective for your codebase. It uses local (Ollama) or cloud AI models to analyze errors, generate surgical patches, and run autonomous improvement pipelines — without ever rewriting your whole file.

**Key idea:** instead of regenerating entire files, the AI outputs only `[SEARCH_BLOCK] → [REPLACE_BLOCK]` patches — precise, reviewable, and safe.

## 📦 Quick Start

### Requirements

- Python **3.11+**
- Windows 10/11, macOS 12+, or Linux Ubuntu 20+

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/signupss/ai-code-sherlock.git
cd ai-code-sherlock

# 2. Install dependencies
pip install -r requirements.txt

# 3. Run
python main.py
```

```

```

## ▶️ Live Script Execution & Interactive Stdin

Click **▶ Run Script** in the toolbar to run the active file (or pick any script). Output streams live into the **📋 Logs** tab — color-coded by stream:

| Color | Meaning |
|-------|---------|
| White | stdout |
| Red | stderr |
| Purple | System events (start / finish) |
| Yellow | User input echo (`>> text`) |

### Interactive stdin

While the script is running, an **input field** activates at the bottom of the Logs tab. Type anything and press `Enter` or click `⏎` to send it to the running process — exactly as if you typed it in a terminal.

Works natively with all language input mechanisms:
- Python `input()`, `sys.stdin.read()`
- Bash `read VAR`
- Node.js `readline` / `process.stdin`
- Ruby `gets`, `$stdin.gets`
- PHP `fgets(STDIN)`
- Go `fmt.Scan`, `bufio.NewReader(os.Stdin)`
- And any other language that reads from stdin

The stdin pipe stays **open for the full duration** of the script. The input field is automatically disabled once the script exits.

### How it works internally

```
Qt main thread          Worker event loop (asyncio)
─────────────────       ──────────────────────────────────
send_stdin("show\n") → asyncio.run_coroutine_threadsafe
                      → queue.put("show\n")
                      → _stdin_pump() drains queue
                      → proc.stdin.write(...)
                      → proc.stdin.drain()
```

The queue-based pump runs as a third coroutine alongside the stdout/stderr readers — **no race conditions, no closed-pipe errors**.

| Language | Extension(s) | Interpreter |
|----------|-------------|-------------|
| Python | `.py` | `python -u` (unbuffered) |
| JavaScript | `.js` `.mjs` `.cjs` | `node` |
| TypeScript | `.ts` | `ts-node` / `npx ts-node` |
| Ruby | `.rb` | `ruby` |
| PHP | `.php` | `php` |
| Perl | `.pl` `.pm` | `perl` |

## ⚙️ Auto-Improve Pipeline

The most powerful feature — runs your script, reads output, generates a patch, validates syntax, applies it, and repeats autonomously.

```
Goal:     "achieve f1 > 0.85 on validation set"
Script:   train_model.py
Strategy: SAFE_RATCHET
Max iter: 20
```

### 8 AI Strategies

| Strategy | When to use |
|----------|-------------|
| 🛡️ Conservative | Only fix errors, minimal changes |
| ⚖️ Balanced | Fix + moderate improvements (default) |
| 🔥 Aggressive | Maximum changes, refactor logic |
| 🧭 Explorer | Different approach every iteration |
| 📈 Exploit | Double down on what already worked |
| 🔒 Safe Ratchet | Apply only if metrics improve |
| 🔬 Hypothesis | Form hypothesis → test → validate |
| 🎭 Ensemble | Generate 3 variants, pick best |

## ✂️ Patch Format

The AI generates structured SEARCH/REPLACE blocks:

```
[SEARCH_BLOCK]
    result = data[0]["value"]
[REPLACE_BLOCK]
    if not data:
        return None
    result = data[0]["value"]
[END_PATCH]
```

**New file creation** — empty SEARCH_BLOCK:

```
Создаю `main.py`:
[SEARCH_BLOCK]
[REPLACE_BLOCK]
import pandas as pd

def main():
    df = pd.read_csv("data.csv")
    print(df.describe())

if __name__ == "__main__":
    main()
[END_PATCH]
```

The engine finds the exact match (or normalized whitespace fallback), validates it's unambiguous, creates a backup, applies the patch, runs syntax check — all before saving.

| 🏆 Best-of-N | Pick response with most valid patches |

## 🔍 Sherlock Mode

Enable the **🔍 Sherlock Mode** toggle to activate root-cause analysis:

```
Input:  error logs + relevant code
Output: hypothesis + minimal patch + confidence score
```

Example output:
```
Clue: data[0] accessed before empty-list guard at line 47
Root cause: function called with empty list on edge case
Confidence: HIGH (92%) — stack trace unambiguous
```
